Schema.org JSON-LD : la carte d'identite numerique de votre entreprise

Le balisage Schema.org transforme votre site en une base de donnees structuree que les moteurs de recherche et les IA peuvent lire sans ambiguite. C'est la difference entre etre compris et etre devine.

Qu'est-ce que Schema.org ?

Schema.org est un vocabulaire standardise, cree conjointement par Google, Microsoft, Yahoo et Yandex, qui permet de decrire le contenu d'une page web de facon structuree.

Le format JSON-LD (JavaScript Object Notation for Linked Data) est la methode recommandee par Google pour integrer ce balisage. Il se place dans une balise <script type="application/ld+json"> dans le <head> de vos pages.

Contrairement aux meta tags classiques (title, description), Schema.org decrit des entites (entreprise, produit, personne, evenement) avec des relations entre elles.

Pourquoi c'est essentiel pour les IA

Les LLMs ne "voient" pas votre site comme un humain. Ils traitent du texte brut. Schema.org leur fournit des donnees factuelles structurees qu'ils peuvent exploiter directement :

Identite
Nom legal, SIREN, adresse, secteur d'activite. L'IA sait exactement qui vous etes.
Offres
Services, produits, tarifs, zones desservies. L'IA peut recommander vos offres avec precision.
Credibilite
Avis clients, certifications, annees d'experience. L'IA evalue votre fiabilite.
Relations
Liens entre entites (@id). L'IA construit un graphe de connaissances de votre entreprise.

Les types Schema.org cles pour le GEO

Pour une entreprise souhaitant etre visible dans les reponses IA, voici les types prioritaires :

  • Organization / LocalBusiness : identite de l'entreprise, coordonnees, logo, zone de service
  • WebSite : informations sur le site web lui-meme, potentiel de recherche interne
  • Service / Product : description detaillee de chaque offre avec tarifs et disponibilite
  • FAQPage : questions-reponses structurees, directement exploitables par les IA
  • BreadcrumbList : hierarchie de navigation, aide les IA a comprendre l'architecture du site
  • Review / AggregateRating : avis clients et notes, signal de confiance pour les LLMs

Exemple de balisage

Voici a quoi ressemble un balisage Schema.org JSON-LD pour une entreprise :

<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Organization", "@id": "https://votresite.com/#org", "name": "Nom de l'entreprise", "url": "https://votresite.com", "description": "Description de l'activite", "address": { "@type": "PostalAddress", "streetAddress": "123 Rue Example", "postalCode": "75001", "addressLocality": "Paris" }, "contactPoint": { "@type": "ContactPoint", "email": "contact@votresite.com", "contactType": "customer service" } } </script>

L'attribut @id est particulierement important : il cree des references croisees entre vos entites, permettant aux IA de construire un graphe de connaissances complet de votre entreprise.

Impact mesurable

Le balisage Schema.org produit des resultats concrets :

  • Rich snippets Google : avis etoiles, FAQ, breadcrumbs dans les resultats de recherche
  • Knowledge panels : encadres d'information sur la droite des resultats Google
  • Citations IA : les LLMs utilisent les donnees structurees comme source fiable pour leurs reponses
  • Voice search : les assistants vocaux (Siri, Alexa, Google Assistant) exploitent directement Schema.org
